# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Development Commands

```bash
# Run tests on multiple Ubuntu versions (22.04, latest)
make test

# Sync dotfiles to devcontainer feature source (creates dotfiles.tar.gz)
make sync

# Create a release with semver tag and push (triggers Docker/feature publishing)
make release VERSION=x.y.z
```

## Project Overview

This repository packages Ubuntu dotfiles as three deliverables:

1. **Standalone dotfiles**: Install shell and tools manually via `./dotfiles/install.sh`
2. **Docker image**: Pre-built images published to `ghcr.io/amc40/ubuntu-dotfiles`
3. **Devcontainer feature**: Installable as a devcontainer feature for VS Code dev containers

All three delivery mechanisms use the same underlying dotfiles and install script.

## Architecture

**Source of truth**: `/dotfiles/` directory contains:
- `install.sh` — Main installation script that sets up zsh, oh-my-zsh, Claude Code, and configuration files
- `.zshrc` — Zsh shell configuration (agnoster theme)
- `.claude/` — Claude Code settings and guidelines

**Devcontainer feature** (`/devcontainer-feature/`):
- Defined in `src/ubuntu-dotfiles/devcontainer-feature.json`
- `src/ubuntu-dotfiles/install.sh` — Feature-specific install wrapper that extracts dotfiles from tar.gz
- `test/ubuntu-dotfiles/test.sh` — Feature tests (validates zsh, oh-my-zsh, agnoster theme, ownership)
- Built tar.gz generated by `make sync` command

**Dockerfile**:
- Builds Docker image using `/dotfiles/install.sh` directly
- Supports build args: `UBUNTU_VERSION` (default: latest), `USERNAME` (default: dev)
- Uses zsh as shell and entry point

## Testing

### Automated
```bash
make test
```

Runs devcontainer CLI tests against both Ubuntu 22.04 and latest. The test suite validates:
- zsh is installed
- oh-my-zsh is installed
- agnoster theme is configured
- Default shell is set to zsh
- .zshrc file ownership is correct

Tests are defined in `/devcontainer-feature/test/ubuntu-dotfiles/test.sh`.

### Manual
Ensure that you're able to run:
1. Run the dotfiles install script locally
2. Build the image and see expected changes when running it

## Release Process

Releases are semver-based. Pushing a tag triggers CI/CD:

```bash
make release VERSION=1.2.3
```

This:
1. Updates `devcontainer-feature.json` version
2. Creates commit and semver tag
3. Pushes to origin

GitHub Actions automatically:
- Tests the feature on multiple Ubuntu versions
- Publishes Docker image to GHCR with tags: `1.2.3`, `1.2`, `1`, `latest`
- Publishes devcontainer feature to the registry
